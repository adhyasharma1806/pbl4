GPU Allocation System

An intelligent web-based system to efficiently manage and allocate limited GPU resources in academic environments using priority-based scheduling and real-time monitoring.

Problem Statement

In academic institutions, GPU resources are limited while multiple students and faculty require them simultaneously for AI and Deep Learning tasks. Manual allocation leads to unfair usage, resource monopolization, and lack of transparency. A centralized automated system is required to ensure fair access, priority-based allocation, monitoring, and logging.

Literature Review

Existing systems like Kubernetes GPU scheduling and cloud allocation platforms provide orchestration but are complex and not tailored for academic labs.

Innovation

A lightweight MERN-stack based system designed specifically for universities with role-based access and priority-based scheduling.

System Methodology

Dataset / Input

User requests, GPU availability, priority level, authentication data.

Architecture

MERN Stack (MongoDB, Express.js, React.js, Node.js) with CUDA Toolkit & NVIDIA Drivers.

System Workflow & Authentication

Secure Login & Role-Based Access

The system implements secure authentication with role-based access control. Users must log in before accessing the platform. Based on their assigned role (Student or Admin), they are redirected to separate dashboards with specific privileges. This ensures controlled access to GPU resources and administrative functions.

Student Login

Students sign in using registered credentials. They can submit GPU requests, view request history, track approval status, and request extensions.

Admin Login

Admins log in through a secured portal with elevated privileges. They can review pending requests, approve/reject allocations, manage extension requests, and monitor GPU utilization.

Session & Security

Authentication uses secure API endpoints with controlled access. Only authorized users can perform privileged operations, preventing misuse of GPU resources.

Complete System Flow

User logs in → Role identified → Dashboard loaded

Student submits GPU request with duration & priority

Admin reviews → Approves / Rejects request

Extension requests follow separate approval workflow

Advanced Extension Workflow

New Feature: Controlled GPU Time Extension

In real-world GPU labs, students often require additional computation time for long-running AI training processes. Instead of allowing automatic renewal, our system introduces an approval-based extension workflow to maintain fairness, transparency, and administrative control over limited GPU resources.

Step 1: Student Requests Extension

The student submits an extension request specifying additional hours. The system flags the request using extensionRequested and stores requested hours securely in the database.

Step 2: Admin Review & Approval

The admin dashboard dynamically detects extension requests and moves them into the "Pending Review" section. The admin can approve or reject based on system load and GPU availability.

Step 3: Controlled Allocation Update

Only after admin approval does the system increase durationHours. Extension flags are reset automatically, ensuring controlled GPU utilization.

Key System Features

Role-Based Access Control

Separate dashboards for Students and Admins with secure authentication and authorization.

Priority-Based Scheduling

Requests are categorized into Low, Medium, and High priority to ensure fair GPU allocation.

Real-Time Monitoring

Live GPU utilization tracking and activity logging for transparency and performance monitoring.

Future Scope

While the current system efficiently manages GPU allocation with approval-based control, it can be further enhanced with intelligent automation, scalability mechanisms, and real-time resource optimization techniques.

Auto GPU Load Balancing

Implement intelligent load-balancing algorithms that automatically assign GPU resources based on real-time utilization, reducing manual intervention and improving performance efficiency.

Email Notifications

Integrate automated email alerts for request approvals, rejections, extension decisions, and GPU release reminders to improve communication between students and administrators.
